    • Define a computer system.

      A computer system is the set of physical elements or hardware that are necessary for the explotation of computer programs or software.

    • Mention and explain the basic functions of a computer.

      • Accept and transmit information: through the input/output devices, the computer receives and communicates with the exterior. This devides are responsible for encoding the input info to a format that the computer understands and encode it to transmit it abroad.
      • Data storage: this function is carried out on storage devices, of which the computer has a great variety, depending on their nature and their specific purpose.
      • Data processing: the main component that performs this function is the processor, that, fundamentally, is in charge of reading a series of instructions from RAM, encoded in a way that it can understand them, and execute them in a pre-established order.

    • Mention the elements of a computer system.


      • Physical elements (hardware): set of physical devices that make up the computer or computer system.
      • Logical elements (software): it is made up of applications and data that exploit hardware resources (Computer programs, system softwate and firmware (the software that handles the BIOS of the computer)).
      • Human component: the users who interact with the computers, and those who develop software to make this interaction possible.

    • Mention and briefly explain the types of computers.


      • A supercomputer is a computer with a high level of performance as compared to a general-purpose computer. These types of environments are often used in research, scientific, or military related environments.
      • Mainframes are high-performance computers with large amounts of memory and processors that process billions of simple calculations and transactions in real time.
      • A server is a computer that accepts and responds to requests made over a network. The device that makes the request, and receives a response from the server, is called a client.
      • A personal computer is a general-purpose computer whose size, capabilities, and original sale price make it useful for individuals, and is intended to be operated directly by an end-user with no intervening computer operator.

    • Mention and briefly explain the generations of computers.


      1. First Generation of Computer: Vacuum tubes and boards. In the mid-1940s, Howard Aiken at Harvard and John von Neumann at Princeton, among others, managed to build calculating machines.
      2. Second Generation of Computer: Transistors. Replaced vacuum tubes, smaller, and consumed less electricity. Communication with these computers is through high-level languages or programming languages.
      3. Third Generation of Computer: Integrated circuits. Emerged with the development of integrated circuits, leading to the invention of the microprocessor.
      4. Fourth Generation of Computer: Characterized by the appearance of the microprocessor. The age of the personal computer arrived.
      5. Fifth Generation of Computer - 2019 - Quantum Computers: IBM presented its first commercial quantum computer, called the IBM Q System One, in 2019.

    • Describe the computer components and know how to classify them.

      Any kind of computer consists of HARDWARE and SOFTWARE. Computer hardware is the collection of physical elements that constitute a computer system, all of which are physical objects that can be touched.
      • Computer case: Contains the central processing unit (CPU) and other essential components.
      • Peripheral device: Hardware device that isn’t required for the computer's primary function. Includes input, output, and input/output devices.
        • Input devices: Used to enter information into a computer system.
        • Output devices: Used to send data from a digital device, such as a computer, to a user or another device.
      • Storage devices: Used to store application and user data, such as hard disk drives and USB flash drives.

    • Explain the Von Neumann architecture.

      The Von Neumann architecture consists of a single, shared memory for programs and data, a single bus for memory access, an arithmetic unit, and a program control unit.


    • Describe the function of every unit and its components.


      • CPU: The central processing unit is the brain of the computer that processes program instructions. In modern computers, the CPU is contained on an integrated circuit chip called microprocessor. Is made up of: the control unit, the Arithmetic logic unit (ALU) and Registers
      • Main Memory Unit: RAM (Random Access Memory) is the hardware in a computing device where the operating system (OS), application programs and data in current use are kept so they can be quickly reached by the device's processor.
      • I/O Unit: The input/output unit communicates the microprocessor with components such as auxiliary memories, external storage units or peripherals.
    • Explain the existing types of buses.


      • The address bus, a one-way pathway that allows information to pass in one direction only, carries information about where data is stored in memory.
      • The data bus is a two-way pathway carrying the actual data to and from the main memory.
      • The control bus holds the control and timing signals needed to coordinate all of the computer’s activities.

    • Mention and briefly explain the types of software.


      • System software: the principal system software is the operating system. It manages the hardware, data and program files, and other system resources and provides means for the user to control the computer, generally via a graphical user interface (GUI).
      • Application software: application software is programs designed to handle specific tasks for users. Example: PowerPoint, Excel…
      • Programming software: programs that are used to write, develop, test, and debug other software, including apps and system software... Example: NetBeans, Eclipse…

  • Explain the operating systems functionalities.


    • It allows the user and other programs to communicate with the hardware, so that they do not have to deal with the complexity of the physical devices.

      The operating system as a translator:

      • Users and programs. It communicates using natural or programming language.
      • Operating system. Translates user commands so that they can be understood by the hardware.
      • Hardware. Accepts commands from the operating system and delivers the result.

    • Manages hardware resources, allocating processors in a controlled manner, through memories, file systems and I/O devices.

      The operating system as administrator:

      • Operating system. It is the orchestra conductor that coordinates all the instruments that make up the orchestra to perform a composition.
      • Hardware resources. These are the instruments that sound in coordination under the conductor's baton. Some are more audible than others, but all are essential to perform a work.

  • Explain the operating systems elements and structure.


    • The kernel. You have access to all the operations allowed by the hardware.
    • The services. The hardware administration is performed based on demand for services, which are requested by applications through a system call interface (APIs such as POSIX or Win32).
    • User and application interfaces. They enable communication between the user and the applications and the computer equipment.

  • Mention and briefly explain the operating systems managers.

    • Interface for applications. A set of interface tools that allow programs to use the hardware resources.

    User interfaces. They provide a command-driven, graphical, or natural language environment for the user to communicate with the operating system.

    • Processes. Planning the use of the processor by the processes.
    • Memory. Allocating or releasing main memory processes.
    • Communication. Enabling communication and synchronisation between processes.
    • I/O (input/output). Controlling the input and output of data from peripheral devices.
    • File system. Managing secondary storage through directories and files.
    • Security. Supervising and monitoring user and application access to the system.

  • Explain what a process is and their possible states.


    • Execution: Running on the processor.
    • Ready: It can be switched to run status if selected by the planner.
    • Blocked: Waiting for a response from another process in order to be able to continue with its execution.

  • Mention and briefly explain the main characteristics of an operating system (target device, control system, price usability…)


    • Target device:
      • Personal computer. Also called desktop, for common tasks such as Internet browsing, office work or leisure activities.
      • Server Optimised for remote user management, network services, security, and business process-oriented tools.
      • Mobile device (smartphone, tablet, etc.). Adapted to consider the hardware resources of mobile devices, such as processor, battery, touch screen, etc.
    • Control system:
      • Embedded. Dedicated to performing control tasks that require extreme reliability for uninterrupted operations in an unattended environment, such as controlling a lift or room temperature.
      • Real-time. Systems that must respond to events within a given time interval, such as a car's airbag control system.
    • Price: in certain professional environments, it is a major constraint on the ability to select an operating system.
    • Usability: through intuitive and comfortable interfaces that allow the user to perform their tasks in a simple and effective way.
    • Efficiency: the operating system must be able to squeeze the maximum out of the computer's hardware resources.
    • Compatibility. Some software is not supported by certain operating systems. The same applies to hardware.
      • Indicates what can and cannot be done with the operating system, such as distribution, modifications, etc.
    • Security: No operating system is immune to a computer attack and its damaging consequences, so it is important that the system incorporates security mechanisms for users and programs accessing the system.
    • Upgrading: A frequently updated operating system is a system that adapts to the new requirements of the software and the hardware, as well as being more robust, reliable, and secure.
    • Support: allows you to solve technical problems or queries quickly and easily.
    • Live version: provides the ability to run the operating system without having to install it on the hard disk.
