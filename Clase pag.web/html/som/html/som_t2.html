GENERACIONES DE LOS ORDENADORES

  1. Primera generación (1940-1959): (ENIAC) estos eran muy grandes y poco óptimos, con un alto consumo energético, está compuesto por tubos de vacío. El UNIVAC 1 es el primer ordenador con fines comerciales. Aparece el primer lenguaje de programación (lenguaje ensamblador).
  2. Segunda generación (1960-1965): empezaron a tener transistores y se vuelven más pequeños y económicos. Aparece el concepto de periféricos.
  3. Tercera generación(1966-1971): Eran monousuarios, concepto de familias de ordenadores, sistema operativo. (IBM)
  4. Cuarta generación(1971-1980): multiusuario y multitarea, pero la interfaz sigue en modo texto. Sistemas operativos: Windows, MAC.
  5. Quinta generación (1981-actual): empezaban a tener ordenadores personales en casa.

TIPOS DE SISTEMAS OPERATIVOS

Se clasifican atendiendo ciertos criterios:

  • Tiempo de respuesta: todo el tiempo que pasa desde que el usuario pide algo hasta que el ordenador se lo da.
    • Procesamiento por lotes: ejecución de trabajos al mismo tiempo, este no tiene un momento específico en el que se cumplan las tareas y las tareas se terminan cuando el equipo es capaz de completarlas.
    • Interactivo: interactúa con el usuario, el tiempo de respuesta es menor, para ello se utiliza el tiempo compartido.
    • Tiempo real: el tiempo de respuesta es prácticamente inmediato. Se utiliza en ordenadores de hospitales, transferencias (bancos).
  • Número de usuarios: pueden ser uno o muchos (multiusuario)
  • Número de procesos: uno o varios (simultáneamente).
  • Número de procesadores: Monoproceso, Multiprocesador (simétricos y asimétricos)
  • Trabajo en red:
    • Centralizados: Máquina aislada de la red
    • En red: Se pueden conectar a la red
    • Distribuido: La red pasa desapercibida pero se pueden compartir los recursos

ESTRUCTURA DE UN SISTEMA OPERATIVO

Se pueden dividir en:

  • Monolíticos: Si falla algo, falla todo.
  • Niveles o capas: Está dividido.
  • Máquina virtual: Simulación de sistema operativo. Es idéntico a la real, multitarea.
  • Cliente/servidor: Software en el que las tareas se reparten entre los proveedores de recursos o servicios, llamados servidores, y los demandantes, llamados clientes.

Memo​ria principal

La memoria principal es esencial para ejecutar programas y almacenar datos temporales.

Gestión de la memoria en monoprogramación y multiprogramación:

  • Protección: Krita utiliza la protección de memoria para evitar que procesos no autorizados accedan a áreas críticas.
  • Particionamiento:
    • Paginación: Krita organiza la memoria física en porciones de igual tamaño para facilitar el manejo de recursos, pero puede haber fragmentación interna.
    • Segmentación: Krita divide la memoria según las necesidades del programa, lo que puede causar fragmentación externa.

JERARQUÍA

Gestión de Entrada y Salida

  • Interrupción: El hardware produce una señal para alarmar al sistema. Si el procesador está haciendo algo, lo interrumpe.
  • DMA (Acceso directo a memoria): Continuación de la actividad a partir de la interrupción.
  • Caching: Se utiliza para almacenar datos en una memoria rápida de manera temporal para que se acceda más rápidamente a ellos.
  • Buffing: Muy parecido al spooling, solo que estos se guardan para que puedan ser procesados más adelante. Su objetivo es evitar pausas en la lectura o escritura de datos.
  • Spooling: Proceso en el que se introducen los programas en una memoria o en un disco de forma que cualquier dispositivo pueda acceder a ellos.

Gestión de Archivos

  • FAT (16,32): Sistema operativo que cuenta con archivos. Las diferencias entre el FAT16 y FAT32 son los nombres de archivos, ya que el 16 no podía pasar el nombre de 8 caracteres.
  • NTFS: Mejora el sistema operativo con seguridad.
  • EXT2,3,4: Sistema operativo de LINUX
    • Journaling: Fue introducido en EXT3, mejora todo y son compatibles entre los tres.

Gestión de Seguridad

  • Seguridad lógica: Una contraseña.
  • Seguridad física: Cableado, cámaras (parte física).
